Peer Review Session

Groups of 3. Each person shares 2 skills.

For each skill, the reviewer:

  1. Reads the skill file (not the output — the skill itself)
  2. Runs it on a test file
  3. Fills in the rubric
  4. Discusses findings

Then revise one skill based on the most impactful feedback.

Score each 1–3. Total: /21.

Write specific feedback for any score of 1 or 2.

The Limitation of Single Skills

One skill does one thing.

A real documentation audit involves:

  1. Coverage check (are all functions documented?)
  2. Completeness check (required sections?)
  3. Style compliance
  4. Readability score
  5. Compile and prioritize findings
  6. Produce an actionable report

A multi-step skill orchestrates all of these.

Skill Maintenance: 4 Rules

  1. Every skill needs a comment header

(version, author, tested-with, last updated)

  1. Version your skills

(Skills degrade as models update)

  1. Document breaking changes

(Output format changes affect downstream processes)

  1. Review skills quarterly

(A skill that worked 6 months ago may be outdated)
