The Hierarchy

Marketplace          ← a catalog (one GitHub repo) listing plugins
  └── Plugin         ← the shareable bundle you install
        ├── Commands ← slash-command skills (/check-style)
        ├── Skills   ← auto-invoked SKILL.md skills
        ├── Agents   ← subagents with own prompt & tools
        ├── Hooks    ← event handlers
        └── MCP servers
A marketplace contains plugins. A plugin bundles commands, skills & agents.

The Four Nouns

TermWhat it isAnalogy
MarketplaceCatalog file (marketplace.json) in a Git repoApp store
PluginInstallable bundle (plugin.json + folders)An app
CommandSlash-command skill — /nameA button
AgentSubagent with own prompt, tools, modelA specialist
Marketplace ≠ plugin. The catalog vs. the thing you install.
6 / 15

Command vs. Skill vs. Agent

  • Command — runs when you type /name. Everything you built in Module 3.
  • Skill — a SKILL.md with a description, so Claude can auto-invoke it.
  • Agent — an isolated worker with its own tools & model, for delegated multi-step jobs.

Namespacing prevents collisions — a check-style command in the doc-skills plugin runs as:

/doc-skills:check-style

Installing — Step 1: Add the Marketplace

/plugin marketplace add AmanProjects/twtai-skill

→ Successfully added marketplace: twtai

Also accepts:

  • owner/repo@v1.0 (branch or tag)
  • a git URL (GitLab, etc.)
  • ./local-folder
A marketplace is just a GitHub repo with a marketplace.json catalog.
8 / 15

Installing — Step 2: Install the Plugin

/plugin install doc-skills@twtai
twtai is the marketplace name, not the repo (twtai-skill).

Step 3 — use it:

/doc-skills:check-style sample-docs/orders-api.md

Or run /plugin → Discover tab → pick → scope (User / Project / Local).

Manage: /plugin list, /plugin disable, /plugin uninstall.

The Two Files That Make It Work

marketplace.json (repo root, in .claude-plugin/) — the catalog:

{
  "name": "twtai",
  "owner": { "name": "Tech Writer's Tribe" },
  "plugins": [
    { "name": "doc-skills", "source": "./plugins/doc-skills",
      "description": "Docs skills from the TWT program." }
  ]
}

plugin.json (in the plugin's .claude-plugin/) — the manifest:

{ "name": "doc-skills", "version": "1.0.0",
  "description": "Documentation skills." }
Only these two go in .claude-plugin/. commands/, skills/, agents/ sit at the plugin root.

Workshop: Publish Your Own

Work in pairs — one publishes, one installs, then switch.

The 5-step loop:

  1. Create a GitHub repo (your-name/my-skills)
  2. Add .claude-plugin/marketplace.json (pick a name)
  3. Add a plugin folder with plugin.json + your skill in commands/
  4. Commit & push
  5. Share two lines:

```

/plugin marketplace add your-name/my-skills

/plugin install my-plugin@my-marketplace

```

Start from the twtai-skill template — don't build from a blank repo.

When a Marketplace Is Overkill

SituationLightweight option
One skill, one person, onceSend the .md; they drop it in .claude/commands/
Your project teamCommit .claude/commands/*.md into the project repo
Connect to a hosted toolCommit a project .mcp.json with a remote URL

Use a marketplace for versioning, many skills, auto-updates, and one-command install at scale.
