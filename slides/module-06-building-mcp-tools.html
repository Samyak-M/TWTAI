Reading the Server Code: Section by Section

// 1. Imports — load the MCP library and file tools
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import * as fs from "fs";

// 2. Create the server — name and version Claude will see
const server = new McpServer({ name: "doc-quality-checker", version: "1.0.0" });

// 3. Register a tool — function Claude can call
server.tool("check_doc_completeness", "description", { filePath: ... }, async ({ filePath }) => {
  // read file, run check, return structured result
});

// 4. Start the server
const transport = new StdioServerTransport();
await server.connect(transport);

The Critical Distinction

Tool (deterministic):                  Skill (AI-generated):

check_doc_completeness("api.md")       /check-api-doc api.md

→ Always: same input = same output     → Varies: same input ≈ similar output
→ Counts sections mechanically         → Interprets sections intelligently
→ No judgment                          → Uses judgment

Best for: counting, computing,         Best for: reviewing, explaining,
          extracting, validating                generating, analyzing
