Transform the content at $ARGUMENTS. (Module 4 — Transformer pattern.)

## From
Expert-level prose: dense, passive, long sentences, undefined jargon.

## To
Beginner-friendly prose: active voice, present tense, short sentences (under 25 words), every technical term defined on first use, second person ("you").

## Preserve (do NOT change)
- Code samples, command names, file paths, and product/proper nouns
- The factual meaning and any numeric values
- Document structure (headings, tables, lists)

## Change
- Passive → active voice
- Future tense → present tense
- Split run-on sentences
- Add a brief inline definition the first time a jargon term appears

## Output
The transformed content only. No comparison, no markup showing changes.
